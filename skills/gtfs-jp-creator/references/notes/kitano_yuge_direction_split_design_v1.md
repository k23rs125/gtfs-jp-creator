# コスモス号 弓削線 2方面分離 設計書 v1（方面見出し行による縦ブロック分割）

対象: `scripts/extract_timetable_coords.py`（Step1 座標方式）
前提メモ: `kitano_coords_block_issue.md`（課題の発見と改良X＝混在の検出・警告まで）
作成日: 2026-06-16
位置づけ: 課題メモ §4「弓削線2方面の本質的分離は未対応」への解決設計。実PDF(kitanoB.pdf page2)の

## 0. 結論（先に）

弓削線2方面は「時刻列が横方向に混在」しているのではなく、**同じ時刻列を共有して、方面ごとに
縦方向の2セクションへ積層**されている。各セクションは PDF 上に**方面見出し行**で明示されている。
よって解は「座標gapで割る」ではなく、**方面見出し行を検出して表をセクション（縦ブロック）に
分割し、セクションごとに既存の列ロジックを適用する**こと。これは座標方式の枠内で実装でき、
インガット号（中段見出しなし）には影響しない。

## 1. 構造の再解明（kitanoB.pdf page2 実測）

弓削線ブロックの時刻列 x（pdfplumber 実測）: 約 986 / 1061 / 1133 / 1209 / 1283。
便名行(top≈208)で各列は 弓削1便 / 弓削3便 / 弓削5便 / 弓削8便 / 弓削10便 に対応。

表は**縦に2セクション**：

| セクション | 見出し(行) | 範囲(top) | 並び |
|---|---|---|---|
| A 生涯学習センター方面 | `【弓削線（生涯学習センター方面）】` top≈160, x=750-1064 | 〜560 | 古賀茶屋駅(8:22…)→…→北野生涯学習センター(8:55…) |
| B 古賀茶屋駅方面 | `弓削線（古賀茶屋駅方面）` top≈592 | 592〜984 | 北野生涯学習センター(9:03…)→…→古賀茶屋駅(9:30…) |

同一列 x=1209（弓削8便）で実測すると:
- A末尾: 北野生涯学習センター **14:49**（top560）
- B先頭: 北野生涯学習センター **13:35**（top648）

→ 現行抽出は1列を上から下へ1便として読むため、A の便と **B の別便**（同じ x に印字された
異なる便）を連結し、`14:49 → 13:35`（−74分）という物理的にありえない逆行を作る。これが
課題メモの「block1 col_x=1209.5 で seq23 14:49 → seq24 13:35」の正体。

**重要な訂正**: 課題メモは「2方面の時刻列が同一ブロックに混在・名前列xも時刻列gapも通常の便間隔と
区別できず座標で分離不能」と結論したが、実体は**縦セクション積層**であり、**方面見出し行という
明示的な区切り**がある。座標(x)だけ見ていたため見落としていた。座標方式に「見出し行の検出」を
足せば分離できる。

## 2. 解法：方面見出し行による縦ブロック分割

### 2-1. 見出し行の検出規則
表本文の各行（y クラスタ）について、次をすべて満たすものを「セクション見出し行」とする:
1. 路線/方面を示す括弧パターンを含む: `【…】` または `（…方面）`／`（…方向）`／`（循環…）`。
2. **時刻セル（`\d{1,2}:\d{2}`）を1つも含まない**。
3. 表本文の y 範囲内にある（最上部のページ見出し・最下部の案内文〔footer〕は除外）。

実測での該当（kitanoB page2）:
- top≈144 `【北野線（循環：古賀茶屋駅／コスモすまいる／大城ますかげ）】`
- top≈160 `【弓削線（生涯学習センター方面）】`
- top≈592 `弓削線（古賀茶屋駅方面）`  ← これが弓削の縦分割点

誤検出に注意すべき行（規則3で除外）:
- 最下部の案内文（例 top≈1040「【運行に関する問い合わせ】…別コース運行…」）は footer。
  y がページ下端側・幅広・問い合わせ等の語、で footer と判定し除外する。
- 見出し行が隣接時刻行と y で近接して1クラスタに混ざると規則2を誤る。**y クラスタは細かめ
  （例 4px）に取り、見出し語を含む行は単独で切り出す**。

### 2-2. 分割アルゴリズム
1. 既存どおり停留所名行・時刻列を検出する。
2. 表本文を、検出した見出し行の top で**縦に区間分割**する（見出し行〜次の見出し行の手前まで＝
   1セクション）。最上部見出しから最初のセクションが始まる。
3. **各セクションの y 範囲内**で、既存の「列ごとに上から時刻を拾って1便にする」処理を行う。
   セクションをまたいで時刻を連結しない（これが逆行の根治）。
4. セクションの見出しテキスト（例「生涯学習センター方面」）を、そのセクションの全便の**方面ラベル**
   として保持し、`trip_headsign` / `direction_id` 付与に使う（Step3 で活用、副次的な精度向上）。

### 2-3. なぜ座標gap案(案2)より良いか
課題メモの案2（時刻列gapで割る）は「弓削2方面の列間隔が通常便間隔と同じ74px」で破綻した。
本案は**列(x)を割るのではなく行(y)をセクションに割る**。区切りは PDF が明示する見出し行なので、
便間隔の曖昧さに依存しない。座標方式の「決定的・誤差ゼロ」を保てる。

## 3. インガット号（jojima）への影響＝回帰安全性

jojimaB.pdf page2 の見出し検出結果: 最上部 top≈152 に方面ラベル（`【…方面】【…方面】`、列の
横並び方向ラベル）と、最下部 top≈1104 の案内文のみ。**表本文の中段に見出し行は無い**。
→ 「中段の見出し行で縦分割」規則は jojima では発火せず、**抽出結果は不変**（regression_snapshot の
`jojimaA/B *_BEFORE.json` とハッシュ一致するはず）。jojima の横並び方面は既存の名前列x方式が処理する。

## 4. 実装方針（extract_timetable_coords.py）

- 追加: `detect_section_headings(chars, body_y_range) -> [(top, text, x0, x1)]`（§2-1の規則）。
- 変更: ブロック生成を「名前列クラスタ × セクション(y区間)」に拡張。各 (名前列, セクション) の
  交差で便を作る。セクション境界をまたぐ時刻連結を禁止。
- 付与: 各便に `direction_heading`（見出しテキスト）を持たせ、JSON 出力に含める。
- フラグ: 改良X の `multi_pattern_suspected`（便内30分逆行）は**保険として残す**。分割後に
  これが発火しなければ分割成功の自動判定に使える（分割後の健全性チェック）。
- `--blocks` 手動指定（改良X）は併存（座標的にどうしても割れない異常PDF向けの最終手段）。

## 5. 検証計画（成功条件）

1. **回帰**: インガット号 A/B を再抽出し `regression_snapshot/*_BEFORE.json` とハッシュ一致
   （誤差ゼロ維持）。中段見出しが無いので一致するはず。
2. **コスモス号B（成功条件）**:
   - 弓削線が「生涯学習センター方面」「古賀茶屋駅方面」の2セクションに分かれる。
   - 同一列 x=1209 の `14:49 → 13:35` 逆行が消える（各セクション内は単調増加）。
   - 弓削線の便で `multi_pattern_suspected` が発火しない。
   - 各便に方面ラベルが付く（弓削1便→生涯学習センター方面 等）。
3. **北野線（循環）**: top≈144 の循環見出しで1セクション。循環の正当な戻り（最大8分程度）は
   セクション内に保持され、誤って分割しない。

## 6. リスクと限界

- 見出しテキストの表記揺れ（全角空白・改行分割）で括弧パターン検出が外れる可能性 → 正規化
  （NFKC・空白除去）してからパターン照合する。
- 見出しが無い／画像化された方面ラベルのPDFには無力（その場合は改良Xの混在検出＋`--blocks`
  手動指定＋「正しく失敗」で人へ）。本案は「見出しがテキストで存在する」PDF に有効。
- A日程(R7kitanoA.pdf)は停留所名が画像化で座標方式の対象外（課題メモ §1）。本案も対象外。
