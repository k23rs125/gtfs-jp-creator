# インガット号 評価分析メモ: 版差と抽出精度の切り分け

## 背景
久留米市コミュニティバス「インガット号」城島地域 A日程について、座標方式
(extract_timetable_coords.py → coords_to_claude.py)でPDFからGTFS-JPを生成し、
公式オープンデータ(正解データ)と trip-aligned 比較を行った。

- 入手PDF: 令和6年10月改正版 (jojimaA.pdf)
- 正解データ: jyojima_R8.7_v2.zip

## 比較結果の推移
| 段階 | マッチ便 | 時刻一致 | 備考 |
|---|---|---|---|
| 初期(要予約含む・先頭時刻マッチ) | 8/17 | 36.26% | 要予約混入でドミノずれ |
| 要予約をpickup_type=2で除外 | 8/17 | 64.31% | 取りこぼし(ナフコ等)残存 |
| 取りこぼし対策+名前正規化 | 10/17 | (マッチ便)100% | 先頭時刻一致便のみマッチ |
| 便マッチを停留所列ベースに改修 | 17/17 | 50.74% | 全便対応・真の姿が判明 |

## 決定的な発見: 便ごとの時刻差が「便内で一定」
全17便を停留所列で対応付け、各便の (当方時刻 − 正解時刻) を全停留所で算出した。

| 便 | 便内の差の種類 |
|---|---|
| R01_0_01,02,06,08 / R01_1_01,02,03,05,07,09 (計10便) | {0} |
| R01_0_03 | {+2} |
| R01_1_04 | {+2} |
| R01_1_08 | {+3} |
| R01_0_05 | {+5} |
| R01_1_06 | {+7} |
| R01_0_07 | {-10} |
| R01_0_04 | {-15} |

**全便で「差の種類が1つだけ」= 便内の全停留所が同一分だけシフト。**

## 結論
1. 便内で差が一定 ⇒ ランダムな読み違いが存在しない。座標方式の抽出は
   入手PDF(令和6年10月版)に対して**時刻誤差ゼロ**。
2. 便単位で異なる一定シフト(0/+2/+3/+5/+7/-10/-15分)は、ダイヤ改正で
   一部の便の発車時刻が変更された典型パターン。
3. よって公式データとの時刻不一致(約49%)は**抽出誤差ではなく、入手PDFと
   公式データのダイヤ版差**に起因する。
4. 時刻の厳密一致評価は、PDFと正解データの版を揃えることが前提条件。
   停留所構造・経路順・停留所名は版差の影響を受けにくく、これらは
   高精度で一致している(停留所名不一致0)。

## 評価方法上の含意
- 便マッチを「先頭の(名前,時刻)」で行うと、版差で先頭時刻がずれた便を
  取りこぼし、見かけの精度が乱れる(8〜10/17)。
- 便マッチを「停留所列」で行うと全便対応でき(17/17)、便ごとの時刻差を
  測ることで「抽出誤差」と「版差」を分離できる。
- この切り分けにより、座標方式の抽出精度の高さ(誤差ゼロ)を実証できた。
