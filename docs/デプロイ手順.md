# デプロイ手順（自治体にURLで提供する）

利用者（自治体職員）が**インストール不要でURLを開くだけ**で使えるように、
アプリをサーバでホスティングする手順です。バス時刻表は公開情報なので、
サーバ運用の秘匿ハードルは低く、**重い依存（Java検証・P11座標・OSRM経路・OCR）を
サーバ側に正しく入れておける＝全員が同じ検証済みの正確なパイプライン**を使えます。

Docker で「Python＋Java＋アプリ」を1つに固めて配置します。

---

## 0. 前提
- サーバに **Docker** が入っていること。
- 検証用 jar が配置済みであること（gitには含まれない）:
  `skills/gtfs-jp-creator/tools/gtfs-validator-cli.jar`
  （無ければ MobilityData の GTFS Validator リリースから CLI jar を入手して配置。README参照）
- サーバから**インターネットに出られる**こと（OSRM公開API・P11取得・地図タイル・CDNに使用）。

---

## 1. ビルド
リポジトリのルートで:
```bash
docker build -t gtfs-jp-app .
```

## 2. 起動（ローカル/サーバ共通）
自動保存とP11キャッシュを残すため、データ用ボリュームをマウントします。
```bash
docker run -d --name gtfs-app --restart unless-stopped \
  -p 8534:8534 \
  -v gtfs_data:/data \
  gtfs-jp-app
```
- `--restart unless-stopped`：サーバ再起動後も自動で立ち上がる。
- `-v gtfs_data:/data`：`/data` に自動保存(`~/.gtfs_jp_app`)等を永続化。

## 3. 動作確認
ブラウザで `http://<サーバのIP>:8534/` を開く → タイトルが出ればOK。
- 初回アクセスでURLに `?sid=...` が付きます（利用者ごとに作業が分離される仕組み）。

---

## 4. 公開ホスティング（HTTPS・独自ドメイン）
社内やインターネットに出すときは、前段に **リバースプロキシ（nginx等）** を置き、
**HTTPS化**と（必要なら）**アクセス制限**をかけます。

nginx 例（`/etc/nginx/sites-available/gtfs`）:
```nginx
server {
    listen 443 ssl;
    server_name gtfs.example.jp;
    ssl_certificate     /etc/letsencrypt/live/gtfs.example.jp/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/gtfs.example.jp/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8534;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;      # StreamlitはWebSocketを使う
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 600s;                     # OCR等の長い処理に備える
    }
}
```
- 証明書は Let's Encrypt（certbot）で取得。
- 限定公開にするなら nginx の Basic認証 や IP制限、または組織のSSOを併用。

---

## 5. 自治体への提供（使ってもらう）
1. 上のURL（例 `https://gtfs.example.jp/`）を担当者に渡す。
2. 利用者は**ブラウザで開くだけ**。インストール・管理者権限・exe警告は不要。
3. **URLをブックマーク**しておけば、タブを閉じても同じURLで**続きから復元**できる
   （利用者ごとに `?sid=` で分離。複数人が同時に使っても衝突しません）。
4. 迷わないよう [利用手順.md](利用手順.md) / [試用ガイド.md](試用ガイド.md) を添える。

---

## 6. 任意設定
- **画像PDFのOCR(MinerU)を有効化**：Dockerfile 末尾の注記どおり
  `RUN pip install -U "mineru[core]"` を足して再ビルド（イメージが数GB増）。
  無い場合、画像PDFは「アプリ内OCR」ボタンが失敗し、手動OCR手順を案内します
  （テキストPDF・Excelはそのまま動作）。
- **Step2をClaudeで自動化（任意）**：`-e ANTHROPIC_API_KEY=xxx` を付けると②の
  Claude構造化が使えます。**未設定でも②はJaccardで自動割当**され動作します。
- **祝日運休**：内閣府の祝日CSVを同梱/取得しておくと祝日展開が使えます。

---

## 7. 運用上の注意
- **リソース**：OCR(MinerU)やP11処理は重い。同時利用が増えるならCPU/メモリを確保。
- **保存の掃除**：`/data/.gtfs_jp_app/session_*.json` が利用者ごとに溜まる。
  定期的に古いものを削除するcron等を用意すると良い（例: 30日以上前を削除）。
- **アクセス制御**：公開URLにする場合は最低限の認証やIP制限を検討。
- **更新**：アプリ更新時は `docker build` → `docker rm -f gtfs-app` → 手順2で再起動。
